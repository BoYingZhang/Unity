
【VR 專案開發紀錄：太空人葆葆肚子餓了】

一、開發條件與平台：
- 開發設備：MacBook M1
- Unity 版本：Unity 2022.3.x (Silicon LTS)
- 測試設備：無 Meta Quest 3，使用 XR Device Simulator 模擬
- 開發目標平台：Meta Quest (Android)
- 最終建置需求：在 Windows 桌機上打包為 .exe

二、開發流程建議：
1. 使用 macOS 進行專案製作與邏輯測試
   - Platform 選 Android（Meta Quest 相容）
   - 使用 XR Device Simulator 測試抓取功能、食物互動流程
   - 模擬器可使用滑鼠與鍵盤操作

2. 專案完成後轉移到 Windows
   - 將整個專案資料夾複製至 Windows 桌機
   - 在 Unity Hub 中打開專案並 Switch Platform → Windows
   - 使用 Build & Run 建出 .exe 執行檔

三、注意事項：
- 使用 URP 或 3D Core，避免 HDRP
- 插件使用 Meta XR All-in-One SDK（跨平台相容）
- 所有資源請置於 Assets 資料夾內，避免硬編路徑
- 專案搬移時請包含 Packages、ProjectSettings 等完整目錄

備註：
- Unity 可隨時 Switch Platform，但轉換期間需重建資源，請耐心等候
- Unity LTS 版本（如 2022.3.x）為 Meta 官方推薦版本，建議開發與打包皆使用此版本
